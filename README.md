1. 프로젝트 개요 – 기획 배경 및 핵심 내용

1.1 프로젝트 기획 배경

영화 관련 서비스는 많지만, 사용자가 실제로 원하는 것은 단순한 영화 정보 조회를 넘어 자신의 취향과 상황에 맞는 추천, 영화 간 비교, 리뷰 요약, 근거 있는 설명까지 한 번에 제공하는 서비스입니다.

기존 방식에서는 사용자가 OTT, 포털, 리뷰 사이트를 각각 찾아다니며 정보를 직접 비교해야 합니다. 또한 일반적인 검색 기반 서비스는 정보는 많지만 정리가 부족하고, 반대로 단순 챗봇은 자연스럽게 답할 수는 있어도 최신성·정확성·근거 제시에 한계가 있습니다.

이 프로젝트는 이러한 불편을 해결하기 위해 기획되었습니다. 사용자가 “오늘 가볍게 보기 좋은 영화 추천해줘”, “라라랜드와 위플래시 중 내 취향엔 뭐가 맞아?”, “인셉션에 대한 전반적인 반응을 요약해줘”처럼 자연어로 질문하면, 에이전트가 질문 의도를 파악하고 관련 지식을 검색한 뒤 구조화된 답변을 제공합니다.

Agent 서비스로 해결하고자 하는 핵심 Pain Point는 다음과 같습니다.

* 영화 선택에 시간이 오래 걸림
* 정보는 많지만 비교와 요약이 어려움
* 추천 결과의 근거가 불명확함
* 취향 기반 질의에 대해 기존 검색 서비스가 충분히 대응하지 못함

이 프로젝트를 시작한 동기는, Bootcamp에서 학습한 Prompt Engineering, Multi-Agent, RAG, Streamlit, LangGraph를 하나의 흐름으로 연결해 실제 사용 가능한 서비스 형태로 구현해보고 싶었기 때문입니다. 영화는 누구나 쉽게 공감할 수 있는 주제이면서도 추천, 검색, 비교, 요약 기능을 모두 담기 좋아 최종 과제 주제로 적합하다고 판단했습니다.

1.2 핵심 아이디어 및 가치 제안(Value Proposition)

본 서비스의 핵심 아이디어는 RAG 기반 영화 전문 Multi-Agent 서비스입니다.
사용자 질문을 하나의 단일 챗봇이 모두 처리하는 것이 아니라, 질문 유형과 목적에 따라 역할이 분리된 에이전트들이 협업하여 더 정확하고 일관된 답변을 생성합니다.

서비스가 제공하는 핵심 기능은 다음과 같습니다.

* 사용자 취향/상황 기반 영화 추천
* 특정 영화 정보 검색
* 두 작품 이상의 비교 분석
* 리뷰/평가 요약
* 추천 근거 및 비교 근거 제시

사용자에게 제공되는 가치와 기대효과는 다음과 같습니다.

* 검색 시간을 줄이고 빠르게 영화 선택 가능
* 단순 목록이 아닌 이유 있는 추천 제공
* 여러 리뷰와 정보를 요약해 한 번에 이해 가능
* 비교 질문에 대해 구조화된 답변 획득 가능

기존 서비스 대비 차별성은 다음과 같습니다.

* 단순 키워드 검색이 아니라 질문 의도 분류 + 검색 + 생성을 결합
* 단일 응답 모델이 아니라 Multi-Agent 구조 기반 역할 분담
* RAG를 통해 검색 결과를 근거로 답변
* Streamlit UI를 통해 실제 서비스처럼 사용 가능
* 향후 MCP/A2A 구조까지 확장 가능한 설계

1.3 대상 사용자 및 기대 사용자 경험(UX)

주요 타겟 사용자는 다음과 같습니다.

* 영화를 추천받고 싶은 일반 사용자
* 여러 영화를 비교해 선택하고 싶은 사용자
* 리뷰를 빠르게 요약해서 보고 싶은 사용자
* 작품 분석이나 큐레이션이 필요한 영화 애호가

사용자에게 제공하고자 하는 경험은 **“검색보다 빠르고, 추천보다 똑똑한 영화 도우미”**입니다.
사용자는 자연스럽게 질문만 입력하면 되고, 서비스는 질문 의도를 해석한 뒤 적절한 에이전트를 거쳐 결과를 제공합니다.

예상 UX 흐름은 다음과 같습니다.

1. 사용자가 자연어로 질문 입력
2. 시스템이 질문 유형을 분류
3. 관련 영화 데이터 및 리뷰를 검색
4. 적절한 에이전트가 답변 생성
5. 사용자는 추천 결과, 비교 결과, 근거 정보까지 한 번에 확인

사용자가 얻는 구체적 Benefit은 다음과 같습니다.

* 영화 선택 시간 단축
* 취향 기반 개인화된 추천
* 복잡한 리뷰 정보의 간결한 요약
* 추천/비교 결과에 대한 신뢰성 향상

2. 기술 구성 – 서비스에 적용할 기술 스택

2.1 Prompt Engineering 전략

본 프로젝트는 역할 기반 프롬프트를 중심으로 설계하였습니다.
각 Agent는 자신의 역할에 맞는 시스템 프롬프트를 가지며, 질문을 추천/정보/비교/리뷰 요약으로 분기하여 처리합니다.

적용 전략은 다음과 같습니다.

* 역할 기반 프롬프트
    * Supervisor Agent: 사용자 질문 의도 분류 및 라우팅
    * Recommender Agent: 취향/상황 기반 추천 생성
    * Info Agent: 영화 기본 정보 및 특징 설명
    * Compare Agent: 두 작품 이상의 차이점 및 공통점 정리
    * Review Agent: 리뷰/평가/반응 요약
* CoT/Few-shot 전략
    * 질문 분류 시 단계적 판단 유도
    * 비교/추천 시 “의도 파악 → 관련 근거 확인 → 응답 생성” 흐름을 내부적으로 유도
    * 추후 대표 질의 예시를 few-shot으로 추가하여 정확도 향상 가능
* 출력 구조화 템플릿 정의
    * 질문 유형
    * 핵심 답변
    * 추천/비교 이유
    * 근거 영화 또는 참고 정보
    * 필요 시 주의사항
* 사용자 유형/상황별 프롬프트 분기
    * 추천형: 분위기, 장르, 감정, 동반자 여부 반영
    * 정보형: 제목, 감독, 배우, 줄거리 등 설명 중심
    * 비교형: 장르, 분위기, 스토리, 추천 대상 기준 비교
    * 리뷰형: 긍정/부정/핵심 특징 중심 요약

2.2 LangChain / LangGraph 기반 Agent 구조

본 프로젝트는 LangGraph 기반 Multi-Agent 구조로 설계하였습니다.
단일 Agent가 모든 기능을 처리하지 않고, Supervisor가 사용자 질문을 해석한 뒤 적절한 전문 Agent로 흐름을 제어합니다.

설계 개념은 다음과 같습니다.

* Supervisor Agent
    * 사용자 질문 의도 분류
    * 다음 실행 Agent 결정
    * 필요 시 재검색 또는 후처리 흐름 제어
* Retriever Agent
    * 벡터 DB에서 관련 영화 정보 검색
    * 질문에 맞는 문서 context 구성
* Recommender Agent
    * 추천 질문 처리
    * 추천 이유와 적합한 사용자 유형 설명
* Info Agent
    * 특정 영화 정보 제공
    * 줄거리, 장르, 감독, 배우 등 설명
* Compare Agent
    * 영화 간 공통점/차이점 비교
    * 상황별 추천 대상 제시
* Review Summary Agent
    * 리뷰 반응 및 평가 요약
    * 전반적 분위기와 특징 정리

Tool Calling과 ReAct 구조는 추후 외부 영화 API 또는 파일 시스템 연동 시 쉽게 확장할 수 있도록 고려하였습니다. 현재 버전에서는 Retriever를 주요 도구처럼 활용하며, Memory는 Streamlit session state 기반으로 사용자 질문 기록을 유지하는 방식으로 반영합니다.

2.3 RAG 구성

본 서비스의 핵심은 RAG 기반 영화 지식 응답입니다.

* 데이터 수집/전처리 파이프라인
    * 영화 제목, 장르, 감독, 출연진, 줄거리, 키워드, 리뷰 요약 데이터 수집
    * 초기 버전은 샘플 영화 데이터셋으로 시작
    * 이후 JSON/CSV/리뷰 문서/PDF 확장 가능
    * 영화 단위 혹은 리뷰 단위로 chunk 분할 가능하도록 설계
* 임베딩 모델 및 Vector DB 선택
    * Azure OpenAI Embedding 모델 사용
    * Vector DB는 Chroma 사용
    * 영화 문서를 임베딩 후 벡터 저장소에 인덱싱
* 검색 로직과 응답 생성 방식
    * 사용자 질문 입력
    * 질문 의도 분류
    * 관련 문서 retrieval
    * 검색된 context 기반 응답 생성
    * context에 없는 정보는 추측하지 않도록 프롬프트 설계

2.4 서비스 개발 및 패키징 계획

* UI 개발 방식
    * Streamlit 기반 UI 구현
    * 사용자가 질문을 입력하고 답변 결과를 바로 확인할 수 있는 구조
    * 검색된 context를 expander로 확인 가능하도록 구성
* BE(API) 및 배포 전략
    * 초기 버전은 Streamlit 단독 실행
    * 확장 버전에서는 FastAPI로 에이전트 로직 분리 가능
    * Docker 기반 배포도 가능하도록 모듈화 고려
* 설정/환경 관리 계획
    * API Key 및 배포 정보는 환경변수로 관리
    * .env 또는 서버 환경변수 방식 적용
    * 코드 내 하드코딩 금지
    * 실무형 구조를 위해 agent / rag / ui / config 모듈 분리 계획

2.5 선택적 확장 기능

* Structured Output / Function Calling
    * 질문 분류 결과를 JSON 형태로 구조화 가능
    * 추천 결과를 카드형 구조로 출력 가능
    * 외부 영화 API 연동 시 function calling 확장 가능
* MCP 기반 파일·시스템·API 연동
    * 향후 영화 데이터 파일, 사내 추천 시스템, 외부 영화 메타데이터 API와 연동 가능
    * MCP 개념을 적용하면 에이전트가 도구와 맥락을 더 표준적으로 활용 가능
* A2A 기반 Agent 협업 구조
    * 추천 Agent와 리뷰 요약 Agent를 분리해 협업 구조 구성 가능
    * 예: 추천 Agent가 작품 후보를 선정하면 Review Agent가 각 작품 반응을 요약하여 보강
    * 더 발전하면 개인화 분석 Agent 추가 가능

3. 주요 기능 및 동작 시나리오

3.1 사용자 시나리오(Use Case Scenario)

사용자 목표는 “내 상황과 취향에 맞는 영화를 빠르고 신뢰성 있게 찾는 것”입니다.

대표 시나리오는 다음과 같습니다.

시나리오 1: 추천 요청

* 사용자는 “주말에 가볍게 보기 좋은 영화 추천해줘”라고 입력한다.
* Supervisor는 이를 추천형 질문으로 분류한다.
* Retriever가 관련 영화 데이터를 검색한다.
* Recommender Agent가 추천 결과와 추천 이유를 생성한다.
* 사용자는 추천 영화와 이유를 확인한다.

시나리오 2: 비교 요청

* 사용자는 “라라랜드와 위플래시 중 뭐가 더 나한테 맞을까?”라고 입력한다.
* 시스템은 비교형 질문으로 분류한다.
* 두 작품에 대한 관련 데이터를 검색한다.
* Compare Agent가 장르, 분위기, 감정선, 추천 대상 기준으로 비교한다.
* 사용자는 자신의 취향에 더 맞는 작품을 선택한다.

시나리오 3: 리뷰 요약 요청

* 사용자는 “인셉션에 대한 전반적인 반응 요약해줘”라고 입력한다.
* Review Summary Agent가 리뷰 관련 context를 기반으로 요약을 생성한다.
* 사용자는 작품에 대한 전반적인 평가 분위기를 빠르게 이해한다.

4. 실행 결과

본 서비스는 Streamlit 환경에서 실행되며, 사용자가 영화 관련 질문을 입력하면 질문 유형을 자동으로 분류하고, 벡터 DB에서 관련 문서를 검색한 후, 그 결과를 바탕으로 답변을 생성합니다.

예상 실행 결과는 다음과 같습니다.

* “인터스텔라 같은 감성적인 SF 영화 추천해줘”
    * 추천형으로 분류
    * 유사한 SF/감성 키워드 기반 영화 검색
    * 추천 결과와 추천 이유 제공
* “기생충 감독이 누구야?”
    * 정보형으로 분류
    * 감독, 장르, 줄거리 등 핵심 정보 제공
* “라라랜드와 위플래시 비교해줘”
    * 비교형으로 분류
    * 장르, 분위기, 추천 대상 기준으로 비교 결과 제공
* “인셉션에 대한 반응을 요약해줘”
    * 리뷰형으로 분류
    * 리뷰 요약 및 작품 특징 정리

6. 프로젝트 수행 소감 / 피드백

이번 프로젝트를 통해 Prompt Engineering, LangGraph, RAG, Streamlit을 하나의 서비스로 연결하는 경험을 할 수 있었습니다. 단순히 모델을 호출하는 수준이 아니라, 실제 사용 가능한 서비스로 만들기 위해서는 질문 분류, 검색 정확도, 응답 구조화, UI 흐름까지 모두 함께 설계해야 한다는 점을 배웠습니다.

특히 Bootcamp에서 배운 내용을 개별 기술로 이해하는 것과, 이를 하나의 End-to-End 서비스로 구현하는 것은 큰 차이가 있다는 점을 느꼈습니다. Multi-Agent 구조를 설계하면서 역할 분리의 중요성을 이해할 수 있었고, RAG를 적용하면서 “모델이 아는 척하는 답변”보다 “검색 결과를 근거로 한 답변”이 훨씬 신뢰성을 높인다는 점도 체감할 수 있었습니다.

또한 Streamlit을 활용해 빠르게 프로토타입을 구현하면서, 사용자 경험과 시각적 전달 방식도 AI 서비스 완성도에 큰 영향을 준다는 점을 알게 되었습니다. 앞으로는 개인화 추천, 실제 외부 API 연동, 리뷰 데이터 확장 등을 통해 더 완성도 높은 영화 에이전트로 발전시키고 싶습니다.
